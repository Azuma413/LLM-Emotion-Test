# LLM Emotion Test
## 概要・計画
AIが根本的に感情表現を持たないのは、学習段階でなんらかの他者とコミュニケーションを取る必要がないからだ。
他人との関係の中でしか感情表現の要求は生まれない。
クオリアの存在が不可思議なのではない。それを観測する主観とでも言うべきものが、この世界で唯一の非科学だ。そしてそれが人間に特有のものであるというのは、少々都合が良すぎる思考だ。ならば、AIのような知性にも、その主観が宿らないと言い切ることはできない。
正直、LLMに感情を持たせるのは簡単だ。学習データを感情別に分類し、それに対してconditioning用のsoft promptやLatent vectorを割り当てれば良い。推論時にそれらを外部からコントロールする事で、感情を再現することは可能だ。
ただし、その感情は内的に生じたものではなく、外部から与えられたものでしかないという点で、この文脈で求められているものとは根本的に異なる。
では、例えば2つのLLMの対話環境を構築し、LLMにはテキストとともに、その感情を制御するためのlatentを出力させる。それを再帰的に入力することで内的に感情を抱かせる。そしてその感情出力をActionとして、2つのLLMが何らかのタスクを達成することを条件に報酬を与え、強化学習を行う。すると、LLMはそのタスク達成に最適な感情表現というものを自然と身につけるのではないか。

具体的には以下のような手順で行う。
まず、Qwen3.5をWRIMEの感情ラベル付きデータセットでSFTする。この時、モデルには、テキスト入力に加えて、learnableなsoft promptを入力する。また、モデルはテキスト出力末尾に、必ず再帰入力用のsoft promptを出力するように学習する。この時、出力soft promptは、全てのsoft promptから一様サンプリングしても良いし、あるいは感情の連続性を表現するために、現在の入力soft promptをそのまま出力させても良い。

次に、SFT後のモデルを学生用、オリジナルを教師用として自己蒸留を行う。
教師モデルには、テキスト入力に加えて、感情表現の指示（怒りっぽく出力して）などを加える。
学生モデルには、テキスト入力に加えて、learnableなsoft promptを入力し、教師モデルの出力latentを教師信号として利用する。その時に、教師側に与えた感情ラベルに応じてsoft promptを切り替える。また、学生モデルはテキスト出力末尾に、必ず再帰入力用のsoft promptを出力するように学習する。
最後に交渉が必要であり、尚且つ最終的なスコアが評価可能なタスクで複数体のLLMを協調動作させ、GRPOで学習する。
これによって、タスク達成に最適な感情表現が学習される。それを観察する。
